5.4 - Internationalization

Supported base Languages

  • English (default)
  • Portuguese (pt-PT)

Setup

Configuration

The configuration is defined in src/i18n.ts.

You should start by defining the supported languages for your app in the resources object:

This takes advantage of JSON module imports to load the translations from the @assets/i18n folder.

The initialization of the i18n object is done with some defaults, including the fallback language and support for some basic HTML nodes. It also loads the LanguageDetector and initReactI18next plugins.

Translation files

The translation files are JSON files located in the /src/assets/i18n folder with the same name as the language code. You only require one file per language.

To improve developer experience we strongly suggest using the i18n Ally VS Code extension. It allows you to edit the translations directly in VS Code, extract keys from hardcoded strings and so on.
